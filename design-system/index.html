Principles

01 · Telemetry, not chrome

Information is the brand.

Data density wins. We use borders to separate, never shadows. Surfaces are 0 px radius. Headlines are condensed and uppercase, numbers are tabular and mono.

02 · F1 red is a signal

Red means now.

Use --accent for the live race, the next event, the active state, the leader's value, and nothing else. Team colors do the team-shading work.

03 · One system, two themes

Light is not an afterthought.

Every token has a paired light-mode value via html.light. Components must render correctly in both — verified pane-by-pane on the Color and Component pages.

Quick start

Every token lives in public/css/app.css. Link it once and you have the entire system. The pre-hydration script in BaseLayout.astro reads localStorage.f1-theme and toggles html.light before paint to avoid a dark→light flash.

/* 1. tokens (read-only, dark default) */ :root { --bg-1: #060709; --fg-1: #f5f5f5; --accent: #E8002D; --f-display: 'Barlow Condensed', sans-serif; } /* 2. opt into light mode */ html.light { --bg-1: #e8eaee; --fg-1: #0d0e12; } /* 3. consume — never hardcode */ .race-card { background: var(--bg-2); border: 1px solid var(--line-1); }

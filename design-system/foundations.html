Display scale

Display sizes drop in big steps — no fine-grained continuum. Every step has a job. Letter spacing is near-zero on big sizes (the condensed face supplies the rhythm) and increases on small uppercase labels.

03 · Surface mechanics

Border-only language

F1gures separates surfaces with 1px borders, not shadows. Three line tokens (--line-1/2/3) escalate from quiet to active. Shadows exist only for floating elements (search palette, nav dropdown, BMC widget).

PANEL
Token--line-1
UseQuiet · default panel border, table row dividers
PANEL · MID
Token--line-2
UseDefault · interactive borders, segmented controls, pills
PANEL · HOVER
Token--line-3
UseActive · hover state, focused inputs (non-error)

Border-radius

0px is the rule. Surfaces have hard corners — the masthead, panels, cards, callouts, pills, tables. The exceptions are deliberate:

RadiusPXWhereRule
None0All surfaces, pills, tables, race cards, panelsDefault. Always.
Round50%Status dots, accent dot, position badge, podium chipFor points-of-light: 6–10px coloured dots
Soft6Listing-card, theme-switcher, sort buttons, listing inputsFLAGGED See Audit · radius — to be unified

Elevation

Shadows are reserved for things that float above the page. Anything else uses borders.

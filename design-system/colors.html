02 / COLOR

Two themes,
one palette.

Every token below is documented in both light and dark on the same row. The dark mode is the canonical surface — light mode is a remap of the same 21-token system. Use only the tokens; never hardcode hex.

07 · Application rules

DO

Use --accent for one thing at a time

On any given screen, accent red marks the single most important active state — next race, leader's points, current nav tab. If it appears twice, the user has nothing to orient to.

DON'T

Don't hardcode hex anywhere

Every value above is a CSS variable. Hardcoding breaks light-mode parity. Exception: #fff on pills with a coloured background (sprint, live), where contrast must hold against either theme.

DO

Layer surfaces by purpose

bg-1 = page, bg-2 = panels/cards, bg-3 = inside-the-panel chrome (table headers, pills, inputs). Don't skip levels — bg-2 on bg-1 is the canonical card; bg-3 on bg-1 reads as broken.

DON'T

Don't tint surfaces with team colour

Team colour is a strip / dot / chip — never a panel background. The .silhouette-photo driver gradient is the only exception (18 % team tint over bg-3).
